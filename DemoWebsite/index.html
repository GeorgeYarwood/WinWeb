WinWeb

A simple HTTP web server

Features:

  • Supports HTTP 1.1
  • Only uses around 1mb of RAM
  • Threaded connections to support multiple clients
  • Support for most common MIME types, can be used as a simple way to deliver content (e.g. Videos, pictures) on your network
  • Simple and lightweight, only 64Kb!

My most recent project has been Bowling Master

It's a peer-to-peer, networked bowling game made using Unity Netcode for GameObjects. To get around NAT/firewalls I made use of Unity Relay. On it's own Relay is a very manual system requring the host to send a generated code to the client, which they then enter and connect with. I was able to automate this process by creating a server that pairs players together. When a player starts seaching for a game, they send a JOIN packet to the server which registers their request and adds it to a queue. Then when another player does the same it looks to see if a pair can be made. If it can, one player is instructed to start a host session. Once the host has a join code it gets sent back to the server, which then forwards it onto the client, instructing them to join it. At this point the server gets out of the way and the game can start as normal. This wasn't the first solution I tried as I originally attempted to create my own UDP relay server, however no matter what I tried the game instances would just not connect. I used Wireshark and saw Netcode sending SMB packets, so there is clearly much more going on than just a simple UDP connection.

I didn't intend to publically release the matchmaking server, however if there is interest in something like this I would be happy to share it.

One of my largest (completed) projects is StarWarp

I was inspired to make StarWarp after playing StarGate for the Atari 2600. I really enjoyed it's simple yet satisfying gameplay and sought to create my own imitation. StarWarp is another Unity project and includes:

  • Text-based dialogue
  • Endless scrolling
  • Boss enemies
  • Controller support
  • Experimental touch/mobile support

I designed the game to be expandable, so if I ever wish to add more levels or enemies I can do so using ScriptableObjects, and perhaps if it ever gets a following I will do so.
